Language & Localization

The Mica Minecraft Launcher renders its interface in 17 languages and picks the right one automatically on first launch. You can also override the detected language at any time from Settings.

How the launcher picks a language

Each time you start the launcher, it resolves the active language in this order:

  1. Your saved override, if you've set one in Settings → Appearance → Language. This always wins.
  2. Your operating system's language, detected via the standard JVM locale. On Windows this is your Region / Language setting; on macOS it's System Settings → Language & Region; on Linux it's the LANG / LC_* environment.
  3. English (United States) as the fallback if the platform reports a non-language locale (e.g. POSIX or C), or if your OS language isn't one of the 17 we ship translations for.
Changing language takes effect after restart

Switching the override in Settings updates the preference immediately, but any screens already on display keep showing the previous language until the launcher restarts. The Settings page itself reminds you of this when you change the dropdown.

Setting a manual override

  1. Open Settings from the navbar (or press Ctrl+, / Cmd+,).
  2. Switch to the Appearance tab.
  3. Pick a language from the Language dropdown. The first entry ("Use OS Language") restores the auto-detection described above and shows which language the OS would resolve to in parentheses.
  4. Click Save. Restart the launcher to see the new language applied to every screen.

Supported languages

The launcher ships translations for the world's 17 most spoken languages plus English:

  • Arabic (عربي)
  • Chinese, Simplified (简体中文)
  • Chinese, Traditional (繁體中文)
  • Dutch (Nederlands)
  • English (source language)
  • French (Français)
  • German (Deutsch)
  • Hindi (हिंदी)
  • Italian (Italiano)
  • Japanese (日本語)
  • Korean (한국어)
  • Polish (Polski)
  • Portuguese, Brazilian (Português do Brasil)
  • Russian (Русский)
  • Spanish (Español)
  • Swedish (Svenska)
  • Turkish (Türkçe)

Each language is identified by its BCP-47 tag (e.g. fr-FR, pt-BR, zh-CN). The list is in SupportedLocales.ENTRIES in the launcher source; the property files live alongside as DisplayStrings_<tag>.properties under src/main/resources/lang/.

Translation quality & reporting issues

Translations for languages other than English are generated by a script that calls Google Translate against the English source bundle, then post-processes parameter placeholders so things like {0} survive translation intact. The output is good but not perfect — in particular, short UI labels (one-word buttons, single nouns) sometimes lose context they had in English.

If you spot a bad translation, the best path is to open an issue on GitHub with the key name (e.g. browse.filter.type.label), your locale, the current rendering, and a suggested correction. Hand-edited corrections survive the next regeneration because the script only fills in missing keys by default.

Adding a new language

If your language isn't in the list above and you'd like to contribute a translation, the contribution path is:

  1. Add a new entry to SupportedLocales.ENTRIES (Java) and TARGET_LOCALES in tools/i18n/translate-locales.js (Node).
  2. Run cd tools/i18n && npm run translate — the script will generate the initial DisplayStrings_<tag>.properties from the English source.
  3. Audit the generated file, hand-correct anything that reads oddly, then open a pull request.

The tools/i18n/translate-locales.js script is incremental: re-running it only translates keys that don't already exist in the target file, so hand corrections are preserved on every subsequent launcher update.
