Modpack Editor

The modpack editor is a built-in tool for creating and editing modpack JSON definition files. It provides a structured interface with validation, search integration, and diff tracking.

Toolbar

  • New — create a blank modpack definition from a template.
  • Open File — load an existing JSON definition from disk.
  • Open URL — fetch and load a definition from a remote URL.
  • Save — write the current definition to a local file.
  • Validate — check the definition for errors (missing fields, invalid URLs, hash mismatches).
  • Diff — compare the current state against the last saved version to see what changed.

Metadata Fields

The top section contains the core modpack information: name, version, description, author, Minecraft version, and Forge version. The Forge version picker lets you browse available Forge builds for the selected Minecraft version.

File Tabs

Files are organized into categorized tabs:

ModsForge mod JARs placed in the mods/ folder.
ConfigsConfiguration files for mods, placed in config/.
ResourcesResource packs and other assets.
ShadersShader packs for OptiFine or Iris.
Initial FilesFiles copied only on first install (e.g., default options.txt).

Each file entry includes a download URL and SHA-1 hash. The editor can auto-calculate hashes by downloading the file and computing its checksum.

Modrinth Search

Use the integrated Modrinth search to find mods by name. Select a result to automatically populate the download URL and hash fields for the correct version.

Version Bumping

The Bump Version button increments the modpack's version number, making it easy to publish updates that the launcher will detect automatically.

URL Checking

The editor can verify that all file download URLs are reachable, helping you catch broken links before publishing.

Tip

Use Validate before saving to catch issues early. The validator checks for required fields, valid URLs, and correct hash formats.
